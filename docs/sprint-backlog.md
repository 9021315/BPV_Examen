# Sprint Backlog

De sprint backlog van dit project wordt bijgehouden in GitHub Projects als Kanban board.

## Project
Veiligheidscheck App

## Kolommen
Ik gebruik de volgende kolommen:

- Todo
- In Progress
- Done

## Werkwijze
Ik plaats alle taken van de examensprint in het Kanban board. Tijdens de sprint verplaats ik taken van Todo naar In Progress en daarna naar Done.

Hiermee bewaak ik mijn voortgang en kan ik aantonen welke werkzaamheden zijn uitgevoerd.

## Taken in de sprint
De belangrijkste taken zijn:

- Wireframes maken
- Projectstructuur opzetten
- Login pagina maken
- Router instellen
- Dashboard medewerker maken
- Login functie bouwen
- Database/backend voorbereiden
- Checklist formulier maken
- Validatie bouwen
- Checklist opslaan
- Bevestiging tonen na opslaan
- Geschiedenispagina maken
- Oude checklists ophalen
- Detailpagina checklist maken
- Admin overzicht maken
- Testplan maken
- Applicatie testen
- Bugs oplossen
- Verbetervoorstellen schrijven
- Presentatie voorbereiden
- Reflectie schrijven

## Al afgerond
De volgende onderdelen zijn al afgerond:

- README maken
- User stories maken
- Acceptatiecriteria maken
- GitHub Project/Kanban board maken
- Het Kanban board
- De taken in Todo, In Progress en Done
- De commits in GitHub
