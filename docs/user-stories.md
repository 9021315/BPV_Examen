# User Stories en Acceptatiecriteria

## User story 1: Inloggen als medewerker
Als medewerker wil ik kunnen inloggen zodat ik toegang krijg tot mijn persoonlijke dashboard.

### Acceptatiecriteria
- De medewerker kan een gebruikersnaam of UID invoeren.
- De medewerker kan een wachtwoord invoeren.
- Bij correcte gegevens wordt de medewerker doorgestuurd naar het medewerker dashboard.
- Bij foutieve gegevens verschijnt er een duidelijke foutmelding.
- De medewerker krijgt alleen toegang tot zijn eigen gegevens.

## User story 2: Inloggen als admin
Als admin wil ik kunnen inloggen, zodat ik alle ingevulde checklists kan bekijken en beheren.

### Acceptatiecriteria
- De admin kan inloggen met admingegevens.
- De admin wordt doorgestuurd naar het admin dashboard.
- De admin kan alle ingevulde checklists bekijken.
- Een normale medewerker kan niet bij het admin dashboard.

## User story 3: Checklist invullen
Als medewerker wil ik een veiligheidschecklist kunnen invullen zodat ik kan vastleggen dat de veiligheidscontrole is uitgevoerd.

### Acceptatiecriteria
- De medewerker kan alle vragen van de veiligheidschecklist invullen.
- Verplichte velden moeten ingevuld zijn.
- Bij lege verplichte velden verschijnt er een foutmelding.
- De ingevulde gegevens zijn duidelijk zichtbaar.

## User story 4: Checklist opslaan
Als medewerker wil ik mijn ingevulde checklist kunnen opslaan zodat de gegevens later opnieuw bekeken kunnen worden.

### Acceptatiecriteria
- De checklist wordt opgeslagen na het klikken op de opslaanknop.
- De checklist wordt gekoppeld aan de juiste medewerker.
- De datum en tijd worden opgeslagen.
- Na het opslaan krijgt de gebruiker een bevestiging.

## User story 5: Geschiedenis bekijken
Als medewerker wil ik mijn eerder ingevulde checklists kunnen bekijken zodat ik mijn eigen controles kan terugzien.

### Acceptatiecriteria
- De medewerker ziet alleen zijn eigen oude checklists.
- De checklists worden overzichtelijk getoond.
- Per checklist is minimaal de datum zichtbaar.
- De medewerker kan een oude checklist openen.

## User story 6: Checklistdetails bekijken
Als medewerker wil ik de details van een oude checklist kunnen bekijken zodat ik kan zien wat ik eerder heb ingevuld.

### Acceptatiecriteria
- De medewerker kan een oude checklist openen.
- Alle ingevulde antwoorden worden getoond.
- De gegevens worden alleen getoond aan de juiste gebruiker.

## User story 7: Admin overzicht
Als admin wil ik een overzicht zien van alle ingevulde checklists zodat ik kan controleren welke veiligheidschecks zijn uitgevoerd.

### Acceptatiecriteria
- De admin ziet alle ingevulde checklists.
- De admin ziet per checklist de medewerker, datum en status.
- De admin kan een checklist openen voor meer details.

## User story 8: Admin detailpagina
Als admin wil ik de details van elke checklist kunnen bekijken zodat ik de ingevulde gegevens kan controleren.

### Acceptatiecriteria
- De admin kan alle antwoorden van een checklist bekijken.
- De gegevens worden overzichtelijk weergegeven.
- Alleen de admin heeft toegang tot deze pagina.
