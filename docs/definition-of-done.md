# Definition of Done

## Betekenis
De Definition of Done beschrijft wanneer een taak of functionaliteit echt klaar is.  
Ik gebruik dit om te controleren of mijn werk voldoet aan de eisen voordat ik een taak naar Done verplaats in mijn Kanban board.

## Een taak is klaar wanneer:

- De functionaliteit werkt zoals beschreven in de user story.
- De acceptatiecriteria zijn gecontroleerd.
- De acceptatietest is uitgevoerd.
- De code netjes en overzichtelijk is geschreven.
- De code past binnen de afgesproken Vue.js en JavaScript structuur.
- De gegevens worden correct opgeslagen en opgehaald.
- De gebruiker krijgt duidelijke meldingen bij fouten.
- De functionaliteit is getest met normale en foutieve invoer.
- Eventuele fouten zijn opgelost.
- De taak is verplaatst naar Done in het Kanban board.
- Er is bewijs gemaakt, zoals een screenshot, commit of korte toelichting.

## Controle voor het afronden

Voordat ik een taak afrond, controleer ik:

1. Werkt de functie zoals bedoeld?
2. Zijn de acceptatiecriteria gehaald?
3. Is de functie getest?
4. Zijn fouten opgelost?
5. Is de code opgeslagen in GitHub?
6. Is de taak bijgewerkt in het Kanban board?
